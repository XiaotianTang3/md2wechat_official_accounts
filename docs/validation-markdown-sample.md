# md2wechat 验证样稿

> 用途：人工验证 Markdown 在 md2wechat 预览、复制到公众号后台、公众号手机预览里的主要排版效果。
>
> 建议测试方式：导入本文件 → 逐个切换「排版」和「配色」→ 复制到公众号后台 → 对照预览检查差异。

# 一级标题：文章主标题

这是一段普通正文。用于检查正文的字号、行距、段间距、颜色、两端对齐和长句换行效果。公众号正文最重要的是长时间阅读不累，所以这里故意放一段稍长的文字，观察在白底环境下是否清晰、稳定、不过度花哨。

这是一段包含 **加粗强调**、*斜体强调*、`inline code`、以及 [普通链接](https://example.com) 的正文。重点检查这些内联元素复制到公众号后是否保留样式，是否影响整段文字的行高。

## 二级标题：结构分节

二级标题通常用于文章中的主要段落。这里检查：

- 标题上下间距是否舒服
- 标题颜色是否体现当前配色
- 标题在公众号后台粘贴后是否变形
- 标题和正文之间是否有足够区分

### 三级标题：局部小节

三级标题用于更细的结构。它不能太抢眼，但也不能和普通加粗混在一起。

#### 四级标题：不常用但需要兜底

四级标题在公众号文章里不一定高频，但 Markdown 用户可能会写。这里主要检查它是否至少可读，不出现明显错乱。

## 特殊字符与中英文混排

AI Agent、Markdown、HTML、CSS、微信公众号、localStorage、inline style，这些词会频繁混排。这里检查英文、中文、代码词、产品名在同一段中的视觉稳定性。

## 结尾小结

**检查重点：**

1. 预览和公众号粘贴结果是否接近
2. 白底下正文是否舒服
3. 标题、引用、表格、代码块是否稳定
4. 不同配色是否只是增强局部元素，而不是干扰阅读
5. 图片是否需要明确从产品范围里暂时排除
