# 微信公众号粘贴人工验证清单

本清单用于验证 md2wechat 的核心承诺：**Markdown 预览保真度**，以及粘贴到**微信公众号后台**后的**富文本可用性**。

GitHub Actions CI 只能证明能安装、能通过 lint 与 build，**不能**证明粘贴后的 HTML 在微信编辑器里仍然可用。请把公众号后台当作外部黑盒，**用手工对照**完成验证。

## Smoke validation 与 Release validation

两种级别都使用同一份样稿：[validation-markdown-sample.md](validation-markdown-sample.md)。

| 级别 | 何时使用 | 排版模板 | 配色方案 |
| --- | --- | --- | --- |
| **Smoke validation** | 日常改动、小 PR 前的快速 sanity | **仅默认**：`minimal`（极简长文，应用默认布局） | **仅默认**：`default`（默认） |
| **Release validation** | 发布前；或改动**渲染**、**剪贴板复制**、**主题**、**文章样式**相关代码后 | **四种全跑**：`minimal`、`starship`（星舰科技）、`studyDaily`（留学日报）、`xinhua`（仿新华社） | 至少 **`default`（默认）** + **一套非默认**（如墨黑灰、星空蓝） |

Release 级别下，建议对每个排版重复：导入 → 切换模板/配色 → 复制 → 三端对照（或至少每种排版对标题、列表、表格、代码块等关键区块粘贴一次）。

**PR 范围说明：** 纯文档 PR **不必**完整跑公众号流程，除非修改了截图或与粘贴相关的说明。若改动影响**渲染**或**剪贴板复制**，至少做一次手动粘贴；需要深度对照时使用本清单。详见 [CONTRIBUTING.md](../CONTRIBUTING.md)。

## 1. 自动检查

```bash
npm run lint
npm run build
```

进入微信公众号手工步骤前，上述两项均应通过。

## 2. 本地启动应用

```bash
npm run dev
```

在终端输出的本地 URL 打开（一般为 `http://localhost:3000`）。

## 4. 导入样稿

将仓库中的 **`docs/validation-markdown-sample.md`**（[在仓库中打开](validation-markdown-sample.md)）载入 md2wechat：可粘贴全文，或在编辑器中打开后复制。

## 5. Smoke validation（最低要求）

1. 完成上文 **1–2** 节。
2. 保持排版为 **minimal（极简长文）**、配色为 **默认（`default`）**。
3. 滚动预览区，确认无明显崩版。
4. 使用「复制到公众号」等操作，粘贴到**微信公众号**图文草稿。
5. 对照 **三处**：
   - md2wechat **预览**
   - 微信公众号 **桌面网页编辑器**
   - 微信公众号 **手机预览**

## 6. Release validation（完整）

在完成 **Smoke** 的基础上：

1. 对 **每种** 排版主题（`minimal`、`starship`、`studyDaily`、`xinhua`）按需重新载入或粘贴样稿，再次复制，并对上述三处做抽查。
2. 在至少一种排版下，切换配色：**默认** + **至少一套非默认**；重复复制与对照。

## 7. 粘贴流程建议

1. 在 md2wechat 中确认预览可接受。
2. 触发复制（inline style HTML + 纯文本 fallback）。
3. 粘贴到微信图文编辑（草稿 / 素材库图文）。
4. **不要**只信 md2wechat——尽量用微信客户端自带的文章预览做最终核对。

<a id="fail-criteria"></a>

## 9. 何种情况算 **Fail**

出现以下情况应判 **Fail**（提 issue、阻塞发布等）：

- 正文在公众号编辑器或手机预览中**不可读**，或间距**明显错乱**。
- **标题层级**丢失，导致文章结构**无法辨认**。
- 粘贴后出现**大面积样式丢失**（非轻微差异）。
- **表格、代码块或列表**出现**明显破版**，且**不能**用 [可接受的已知限制](#known-limitations) 解释。

<a id="known-limitations"></a>

## 10. 可接受的已知限制

在仍符合产品意图的前提下，以下**不**自动算作失败：

- 微信编辑器在粘贴后可能**清洗或改写**部分样式。
- **远程图片**可能无法随粘贴保留，或在微信中表现不同；不要过度承诺图片能力。
- **本地图片**需要时应在微信编辑器中上传。
- **宽表**可能需要在微信中手工调整宽度或换行。
- **富文本剪贴板**行为因浏览器、操作系统而异。

更多说明见 [CONTRIBUTING.md](../CONTRIBUTING.md) 中「微信公众号后台粘贴行为」一节。

## 11. 公开仓库中的截图

若在 PR 或 issue 中附带截图：

- 正文内容请仅使用 **[validation-markdown-sample.md](validation-markdown-sample.md)** 中的样稿。
- 对账号名、草稿标题、私有内容、登录态等敏感信息打码或裁剪。

## 12. 哪些无法被清单或 CI 替代

以下仍需人工与真实环境：

- 真实微信公众号后台的粘贴结果。
- 各浏览器剪贴板差异。
- 微信客户端内的手机预览。
- 主观上的可读性是否「够好」。

本仓库默认**不**引入自动化浏览器测试或图片快照测试，除非维护者后续另有决定。
